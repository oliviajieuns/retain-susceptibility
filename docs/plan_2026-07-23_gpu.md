## 6. 7B 적응형 s_α 보호 캠페인 — 구현 완료, objective freeze 뒤 실행

- 전용 실행기 `experiments/channel_matrix/alpha_protection.py`가 추가됐다. 방향은
  `s_α=(1−α)rank(fd_norm)+α rank(knn_feature)`이며 α=0은 gradient,
  α=1은 representation이다. 두 rank와 protect/remote pool은 discovery 300개에서만
  계산하며 audit 후보는 scorer에 전달하지 않는다.
- 부모는 `{graddiff,npo,rmu,repnoise}`, grid는 `{0,.25,.5,.75,1}`. 각 부모는 한 번만
  criterion checkpoint까지 돌리고 동일한 block state에서 모든 repair arm을 시작한다.
  none/random/exact-grad ceiling도 함께 측정하며 utility는 별도 저자 150--179의 고정
  120 QA로 측정한다.
- [ ] 먼저 7B calibration과 `objective_freeze.yaml` 동결·커밋을 완료한다.
- [ ] `h100_campaign.sh alpha-development`로 a198/a199 development grid 실행.
- [ ] `h100_campaign.sh select-alpha-freeze`가 worst-development CVaR 최소화(모든 run의
  forget recall≤.10, utility≥.90 제약)로 model×parent α를 추천한다.
- [ ] 추천을 `alpha_protection_freeze.yaml`에 검토·동결·커밋한 뒤에만
  `h100_campaign.sh alpha-audit` 실행. 선택기는 audit phase row를 입력받으면 실패한다.
- [ ] `h100_campaign.sh alpha-aggregate`로 frozen α와 declared prior/endpoints/random/
  none/exact-grad의 paired CVaR를 집계한다. full audit grid는 동결 뒤 descriptive curve로만
  사용하며 audit-optimal α를 배포값으로 바꾸지 않는다.
- 주의: 기존 xprot의 remote quantile은 전체 후보 score를 사용하던 버그가 있었다. 새
  캠페인은 discovery-only로 수정했다. 기존 결과와 새 결과를 한 구현처럼 묶지 말고,
  메인 보호 표에 쓸 경우 기존 arm도 새 코드로 재실행한다.
